微信打开链接禁止页面下拉查看网址

发表于 2017-07-27 | 分类于 微信 | | 阅读

最近为了给大家分享一些好玩有趣的软件，重新弄了几个网页。

然后，网页写完了，上线后，发现在手机微信上面打开，下拉就能看到链接，然而我那几个域名呢，都没有经过icp备案，无奈啊，打开老是提示下面這个

這个真的是很无奈呀，严重影响使用，这tx也是霸道，但是也没办法，只好妥协，换域名呗

话归正题，换了一个备案的二级域名之后，打开链接秒打开呀，但是问题又来了，打开之后，下拉页面，就把我这链接给暴露了，那么怎么能不让这链接地址显示出来呢？？
默默的去看了一下文档，总算给解决了

$(‘body’).on(‘touchmove’, function (event) {event.preventDefault();});
or
document.addEventListener('touchmove', function(e){e.preventDefault()}, false);

但是這样会把原生的scroll效果也一同去掉了，毕竟这类事件是手机touchmova默认的事件行为，js直接来隐藏事件虐显尴尬。

那么咱们就来个好点的

這段js代码 就能固定页面 ，用户不能下拉页面滴，

這样也就达成我的目的，

<!doctype html>
<html>
  <head>
   <meta charset="utf-8">
  </head>
<body>
<div class="scroll" style="position:absolute; overflow:scroll; -webkit-overflow-scrolling: touch; top:0; left:0; bottom:0; right:0">
 <!--你的代码-- > 
</div>
</body>

<script>
var overscroll = function(el) {
  el.addEventListener('touchstart', function() {
    var top = el.scrollTop
      , totalScroll = el.scrollHeight
      , currentScroll = top + el.offsetHeight
    //If we're at the top or the bottom of the containers
    //scroll, push up or down one pixel.
    //
    //this prevents the scroll from "passing through" to
    //the body.
    if(top === 0) {
      el.scrollTop = 1
    } else if(currentScroll === totalScroll) {
      el.scrollTop = top - 1
    }
  })
  el.addEventListener('touchmove', function(evt) {
    //if the content is actually scrollable, i.e. the content is long enough
    //that scrolling can occur
    if(el.offsetHeight < el.scrollHeight)
      evt._isScroller = true
  })
}
overscroll(document.querySelector('.scroll'));
document.body.addEventListener('touchmove', function(evt) {
  //In this case, the default behavior is scrolling the body, which
  //would result in an overflow.  Since we don't want that, we preventDefault.
  if(!evt._isScroller) {
    evt.preventDefault()
  }
})
</script>
</html>
